🎭
Lý do 1 — Dữ liệu quan hệ nhiều-nhiều (Many-to-Many)
▼

Một bộ phim có nhiều thể loại (Action, Comedy, Thriller) và một thể loại có nhiều phim. Nếu lưu genres vào cột trong bảng movies:

BAD — Lưu genres dạng string trong Movie
# ❌ Sai — không thể query hiệu quả
movie.genres = "Action, Comedy, Thriller"

# Muốn lấy tất cả phim Action thì phải LIKE '%Action%'
# → Không dùng index được → Scan toàn bộ bảng → Chậm
Movie.objects.filter(genres__icontains='Action')  # ❌
GOOD — M2M through table
# ✓ Đúng — query có index, cực nhanh
Movie.objects.filter(genres__slug='action')  # ✓
Movie.objects.filter(genres__slug='action', countries__country_code='KR')  # ✓

Tương tự với: diễn viên (1 người đóng nhiều phim), quốc gia, studio, tags.

Lý do 2 — Phân biệt loại nội dung (medium_type)
▼

Kế hoạch gốc không phân biệt Movie vs Series vs Anime vs OVA. Điều này ảnh hưởng trực tiếp đến UI:

Vấn đề thực tế: Phim lẻ cần ẩn danh sách tập. Series cần dropdown chọn mùa. Anime Movie khác Anime Series. OVA cần liên kết về series cha. Nếu không có trường medium_type, frontend không biết hiển thị gì.

🌍
Lý do 3 — Phụ đề, trailer, diễn viên là tính năng cốt lõi
▼

Một website phim chuyên nghiệp cần:

  • Subtitles: Mỗi tập có thể có Vietsub, Engsub, Korsub. Player cần load đúng file .vtt theo ngôn ngữ người dùng chọn.
  • Trailers: Official Trailer, Teaser, Behind the Scenes — mỗi cái là một bản ghi riêng.
  • Cast & Crew: Người dùng click vào tên đạo diễn → xem tất cả phim của họ. Không thể làm điều này nếu chỉ lưu tên đạo diễn dạng text.
  • Seasons: Kế hoạch gốc thiếu season_number trong Episode → không handle được series nhiều mùa.

NHẬN XÉT

Đánh Giá Kế Hoạch Gốc

✓ Điểm Tốt

  • Stack kỹ thuật đúng: Django + PostgreSQL + Caddy
  • HLS byte-range (single_file) giảm số request, phù hợp LAN
  • Tự động hóa xử lý video bằng subprocess — tư duy đúng
  • Phân chia 7 giai đoạn rõ ràng, logic và tuần tự
  • Dùng Waitress thay vì Django dev server — an toàn
  • Có logging, có backup — biết lo xa

✗ Điểm Đã Sửa

  • Models quá đơn giản — đã nâng lên 14 bảng chuẩn hóa
  • Thiếu medium_type → không phân biệt Movie/Series/Anime
  • Thiếu season_number → không handle series nhiều mùa
  • Thiếu subtitles, trailers, cast/crew, studios
  • Thiếu giải thích kiến trúc hệ thống — đã thêm đầy đủ
  • processor.py cần update để dùng models mới

Tải bản mới nhất từ postgresql.org/download/windows. Trong quá trình cài:

Sau khi cài, mở pgAdmin 4 tạo database:

SQL — pgAdmin Query Tool
-- Tạo user riêng (nguyên tắc least privilege)
CREATE USER film_user WITH PASSWORD 'strong_password_here';

-- Tạo database
CREATE DATABASE filmsite_db
    WITH OWNER film_user
    ENCODING 'UTF8'
    LC_COLLATE 'en_US.UTF-8'
    TEMPLATE template0;

GRANT ALL PRIVILEGES ON DATABASE filmsite_db TO film_user;
Tại sao tạo user riêng? Nguyên tắc "least privilege" — film_user chỉ có quyền trên filmsite_db. Nếu credentials bị lộ, thiệt hại được giới hạn tối đa.

films/models.py — Phần 2: Lookup Tables
class Country(models.Model):
    """ISO 3166-1 alpha-2: US, KR, JP, VN…"""
    country_code = models.CharField(max_length=2, primary_key=True)
    country_name = models.CharField(max_length=100)
    class Meta: db_table = 'countries'
    def __str__(self): return self.country_name


class Genre(models.Model):
    genre_name  = models.CharField(max_length=100, unique=True)
    slug        = models.SlugField(max_length=120, unique=True, blank=True)
    description = models.TextField(blank=True)
    class Meta: db_table = 'genres'
    def save(self, *args, **kwargs):
        if not self.slug: self.slug = slugify(self.genre_name)
        super().save(*args, **kwargs)
    def __str__(self): return self.genre_name


class Tag(models.Model):
    tag_name = models.CharField(max_length=100, unique=True)
    slug     = models.SlugField(max_length=120, unique=True, blank=True)
    class Meta: db_table = 'tags'
    def __str__(self): return self.tag_name


class Studio(models.Model):
    studio_name  = models.CharField(max_length=255)
    logo_path    = models.CharField(max_length=500, blank=True)
    country_code = models.CharField(max_length=2, blank=True)
    website      = models.URLField(blank=True)
    class Meta: db_table = 'studios'
    def __str__(self): return self.studio_name


class Person(models.Model):
    full_name     = models.CharField(max_length=255)

3.1 — Lệnh FFmpeg chi tiết
▼
Shell — Lệnh FFmpeg giải thích từng flag
ffmpeg \
  -i input.mp4 \                  # File đầu vào
  -c:v libx264 \                  # Video codec: H.264
  -profile:v baseline \          # Profile: tương thích tối đa
  -level 3.0 \                   # Level 3.0 = max 720p@30fps
  -c:a aac \                     # Audio codec: AAC (HLS chuẩn)
  -b:a 128k \                    # Audio bitrate: 128kbps
  -movflags +faststart \        # Cho phép stream trước khi tải xong
  -start_number 0 \             # Segment đầu tiên đánh số từ 0
  -hls_time 10 \                # Mỗi segment = 10 giây
  -hls_list_size 0 \            # Giữ tất cả segments trong .m3u8
  -hls_flags single_file \      # Byte-Range: gộp thành 1 file .ts
  -f hls \
  output.m3u8
Byte-Range HLS (single_file) là gì? Thay vì tạo hàng trăm file .ts nhỏ, FFmpeg tạo 1 file output.ts lớn + file output.m3u8 chứa "map" offset byte từng segment. Khi cần giây 30-40, trình duyệt chỉ tải byte 4MB→8MB, không tải toàn bộ. Kết quả: ít file hơn, dễ quản lý, phù hợp mạng LAN tốc độ cao.
Thông sốÝ nghĩaThay đổi khi...
-hls_time 10Mỗi chunk = 10 giâyMạng chậm → tăng 30s
-profile baselineTương thích tối đaChỉ Chrome → đổi main/high
-b:a 128kAudio bitrateMuốn âm thanh tốt hơn → 192k

def get_duration_minutes(path: Path) -> int:
    """Lấy thời lượng (phút) bằng ffprobe."""
    r = subprocess.run([
        'ffprobe', '-v', 'quiet', '-print_format', 'csv=p=0',
        '-show_entries', 'format=duration', str(path)
    ], capture_output=True, text=True)
    try: return max(1, int(float(r.stdout.strip()) / 60))
    except: return 0


def encode_hls(input_path: Path, output_dir: Path) -> bool:
    output_dir.mkdir(parents=True, exist_ok=True)
    cmd = [
        'ffmpeg', '-i', str(input_path),
        '-c:v', 'libx264', '-profile:v', 'baseline', '-level', '3.0',
        '-c:a', 'aac', '-b:a', '128k', '-movflags', '+faststart',
        '-start_number', '0', '-hls_time', '10',
        '-hls_list_size', '0', '-hls_flags', 'single_file',
        '-f', 'hls', str(output_dir / 'output.m3u8'),
    ]
    try:
        result = subprocess.run(cmd, capture_output=True, text=True, timeout=3600)
        if result.returncode != 0:
            log.error(f"FFmpeg lỗi: {result.stderr[-500:]}")
            return False
        return True
    except subprocess.TimeoutExpired:
        log.error(f"Timeout: {input_path.name}"); return False


def parse_filename(filename: str):
    """
    ten-phim_tap-3.mp4      → slug='ten-phim', season=1, ep=3
    ten-phim_s2_tap-5.mp4   → slug='ten-phim', season=2, ep=5
    """
    stem = Path(filename).stem
    season = 1
    if '_s' in stem and '_tap-' in stem:
        parts  = stem.split('_s')
        s_ep   = parts[1].split('_tap-')
        return parts[0], int(s_ep[0]), int(s_ep[1])
    if '_tap-' in stem:
        parts = stem.split('_tap-')
        return parts[0], 1, int(parts[1])
    return stem, 1, 1   # Phim lẻ


def process_video(mp4_path: Path):
    movie_slug, season_num, ep_num = parse_filename(mp4_path.name)

    movie, created = Movie.objects.get_or_create(
        slug=movie_slug,
        defaults={
            'title'      : movie_slug.replace('-', ' ').title(),
            'medium_type': 'Movie' if ep_num == 1 else 'Series',
            'status'     : 'Completed',
        }
    )
    if created: log.info(f"Tạo Movie: {movie.title}")

    # Bỏ qua nếu tập đã tồn tại
    if Episode.objects.filter(movie=movie, season_number=season_num, episode_number=ep_num).exists():
        log.info(f"Bỏ qua (đã tồn tại): {mp4_path.name}")
        return

    output_dir = HLS_DIR / movie_slug / f's{season_num}-ep{ep_num}'
    m3u8_path  = f'hls_streams/{movie_slug}/s{season_num}-ep{ep_num}/output.m3u8'

    episode = Episode.objects.create(
        movie=movie, season_number=season_num, episode_number=ep_num,
        m3u8_path=m3u8_path,   # ★ 'video_url' → 'm3u8_path'
    )

    duration = get_duration_minutes(mp4_path)   # ★ phút thay vì giây

    if encode_hls(mp4_path, output_dir):
        episode.duration = duration
        episode.save()
        shutil.move(str(mp4_path), str(DONE_DIR / mp4_path.name))
        log.info(f"✓ {mp4_path.name} → S{season_num}E{ep_num}")
    else:
        episode.delete()   # Xóa record nếu encode thất bại
        log.error(f"✗ Encode thất bại: {mp4_path.name}")


def main():
    mp4_files = list(RAW_DIR.glob('*.mp4'))
    if not mp4_files: log.info("Không có file .mp4 mới."); return
    log.info(f"Tìm thấy {len(mp4_files)} file mp4")
    for f in sorted(mp4_files):
        process_video(f)

if __name__ == '__main__':
    main()

import os, sys, time, requests, logging
from pathlib import Path

sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings')
import django; django.setup()

from films.models import Movie, Genre, Country, Person, MovieCastCrew

log = logging.getLogger(__name__)
JIKAN_BASE = "https://api.jikan.moe/v4"
RATE_DELAY = 0.5     # Jikan giới hạn ~3 req/giây


def jikan_get(title: str) -> dict | None:
    time.sleep(RATE_DELAY)
    try:
        r = requests.get(f"{JIKAN_BASE}/anime",
            params={'q': title, 'limit': 1}, timeout=10)
        r.raise_for_status()
        data = r.json().get('data', [])
        return data[0] if data else None
    except Exception as e:
        log.warning(f"Jikan lỗi khi tìm '{title}': {e}")
        return None


def download_image(url: str, save_path: Path) -> bool:
    if save_path.exists(): return False
    try:
        time.sleep(RATE_DELAY)
        r = requests.get(url, timeout=15)
        r.raise_for_status()
        save_path.parent.mkdir(parents=True, exist_ok=True)
        save_path.write_bytes(r.content)
        return True
    except Exception as e:
        log.warning(f"Tải ảnh lỗi: {e}")
        return False


def enrich_movie(movie: Movie):
    log.info(f"Crawl: {movie.title}")
    data = jikan_get(movie.title)
    if not data: return

    # Cập nhật text fields
    movie.description   = data.get('synopsis', '') or movie.description
    movie.original_title= data.get('title_japanese', '')
    movie.release_year  = (data.get('aired', {}).get('prop', {})
                              .get('from', {}).get('year'))
    movie.total_episodes = data.get('episodes')
    movie.status = 'Completed' if data.get('status') == 'Finished Airing' else 'Ongoing'
    movie.save()

    # Genres (M2M)
    for g in data.get('genres', []):
        genre, _ = Genre.objects.get_or_create(genre_name=g['name'])
        movie.genres.add(genre)

    # Tải poster
    poster_url = data.get('images', {}).get('jpg', {}).get('large_image_url')
    if poster_url:
        save_path = Path(f'media/posters/{movie.id}.jpg')
        if download_image(poster_url, save_path):
            movie.poster_path = f'/media/posters/{movie.id}.jpg'
            movie.save()

    log.info(f"✓ Hoàn thành: {movie.title}")


def main():
    movies = Movie.objects.filter(description='')
    log.info(f"Crawl cho {movies.count()} phim chưa có description")
    for movie in movies:
        enrich_movie(movie)

if __name__ == '__main__':
    main()

def home(request):
    movies = Movie.objects.filter(is_hidden=False).prefetch_related('genres', 'countries')

    # Filters
    genre_slug   = request.GET.get('genre', '')
    country_code = request.GET.get('country', '')
    medium_type  = request.GET.get('type', '')
    query        = request.GET.get('q', '')
    year         = request.GET.get('year', '')

    if genre_slug:   movies = movies.filter(genres__slug=genre_slug)
    if country_code: movies = movies.filter(countries__country_code=country_code)
    if medium_type:  movies = movies.filter(medium_type=medium_type)
    if query:        movies = movies.filter(title__icontains=query)
    if year:         movies = movies.filter(release_year=year)

    # Sort
    sort = request.GET.get('sort', '-created_at')
    if sort in ('-imdb_rating', '-view_count', '-release_year', '-created_at'):
        movies = movies.order_by(sort)

    paginator = Paginator(movies.distinct(), 24)
    page_obj  = paginator.get_page(request.GET.get('page'))

    return render(request, 'films/home.html', {
        'page_obj' : page_obj,
        'genres'   : Genre.objects.all(),
        'countries': Country.objects.all(),
    })


def movie_detail(request, slug):
    movie = get_object_or_404(Movie, slug=slug, is_hidden=False)

    # Prefetch episodes + subtitles (tránh N+1 query)
    episodes = movie.episodes.prefetch_related('subtitles')

    # Tập mặc định: season + episode từ query params
    s   = int(request.GET.get('s', 1))
    ep  = int(request.GET.get('tap', 1))
    current = (episodes.filter(season_number=s, episode_number=ep).first()
               or episodes.first())

    # Group episodes by season
    seasons = {}
    for e in episodes:
        seasons.setdefault(e.season_number, []).append(e)

    return render(request, 'films/detail.html', {
        'movie'  : movie,
        'seasons': seasons,
        'current': current,
        'cast'   : movie.moviecastcrew_set.select_related('person').filter(role='Cast')[:10],
        'trailers': movie.trailers.all(),
    })


def person_detail(request, slug):   # ★ MỚI

        cast_crew=person, is_hidden=False

    const hls = new Hls({ maxBufferLength: 60, maxMaxBufferLength: 120 });
    hls.loadSource(m3u8Url);
    hls.attachMedia(video);
    hls.on(Hls.Events.MANIFEST_PARSED, () => video.play().catch(() => {}));
    hls.on(Hls.Events.ERROR, (_, data) => {
      if (data.fatal) console.error('HLS lỗi:', data.type, data.details);
    });
  } else if (video.canPlayType('application/vnd.apple.mpegurl')) {
    video.src = m3u8Url;   // Safari — native HLS
  }
</script>
hls.js vs video.js: hls.js thuần HLS, nhỏ hơn (~150KB vs ~450KB). Với yêu cầu chỉ phát HLS và subtitle .vtt, hls.js là lựa chọn tối ưu. Safari hỗ trợ HLS native — không cần thư viện.

6.2 — Caddyfile hoàn chỉnh với Security Headers
▼
C:\Caddy\Caddyfile
# Thay 192.168.1.x bằng IP tĩnh của máy bạn
localhost:80, 192.168.1.x:80 {

    # ── Security headers ───────────────────────────
    header {
        X-Content-Type-Options  "nosniff"
        X-Frame-Options         "SAMEORIGIN"
        Referrer-Policy         "strict-origin-when-cross-origin"
        -Server
    }

    # ── Static files (CSS, JS, fonts) ──────────────
    route /static/* {
        header Cache-Control "public, max-age=86400"
        file_server { root C:\Projects\filmsite }
    }

    # ── Media files (HLS, posters) ─────────────────
    route /media/* {
        # m3u8 manifest: không cache (có thể update tập mới)
        @m3u8 path *.m3u8
        header @m3u8 Cache-Control "no-cache, no-store"

        # .ts segments: cache 7 ngày (nội dung bất biến)
        @ts path *.ts
        header @ts Cache-Control "public, max-age=604800"

        # .vtt subtitles: cache 1 ngày
        @vtt path *.vtt
        header @vtt Cache-Control "public, max-age=86400"

        # Ảnh poster/backdrop: cache 7 ngày
        @img path *.webp *.jpg *.png
        header @img Cache-Control "public, max-age=604800"

        file_server { root C:\Projects\filmsite }
    }

    # ── Django App ─────────────────────────────────
    reverse_proxy localhost:8000 {
        header_up X-Forwarded-For   {remote_host}
        header_up X-Forwarded-Proto {scheme}
    }

    # ── Logging ────────────────────────────────────
    log {
        output file C:\Caddy\access.log {
            roll_size 10mb
            roll_keep 3
        }
        format json
    }
}
Tại sao .m3u8 không cache nhưng .ts thì cache 7 ngày? File .m3u8 là manifest — danh sách segment. Nếu cache manifest mà bạn thêm tập mới, trình duyệt vẫn đọc bản cũ. File .ts là nội dung video thực sự — một khi đã encode xong, không bao giờ thay đổi, cache mạnh an toàn và tăng hiệu suất đáng kể.
